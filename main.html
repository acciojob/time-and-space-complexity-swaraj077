Time & Space Complexity

Time complexity is defined as the amount of time taken by an algorithm to run, as a function of the length of the input.
It measures the time taken to execute each statement of code in an algorithm.
Time complexity can make the program behave in required optimal conditions, and by doing so, it makes us efficient programmers.

The Space Complexity of an algorithm is the total space taken by the algorithm with respect to the input size.
Space Complexity includes both Auxiliary space and space used by input.
For example, if we want to compare standard sorting algorithms on the basis of space, then Auxiliary Space would be a better criterion than Space Complexity.
Merge Sort uses O(n) auxiliary space.
